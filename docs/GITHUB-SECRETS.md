# AION Vision Hub — GitHub Secrets & Variables Reference

Last updated: 2026-03-18

This document lists every secret and variable required by the CI/CD pipelines defined in `.github/workflows/`. Use it when setting up a new repository fork, migrating to a new GitHub organization, or rotating credentials.

---

## How to Set Secrets and Variables

### Secrets (encrypted, write-only)

1. Go to **GitHub Repository** > **Settings** > **Secrets and variables** > **Actions**
2. Click **New repository secret** (or **New environment secret** for environment-specific)
3. Enter the **Name** and **Value** exactly as listed below
4. Click **Add secret**

### Variables (plain text, readable in logs)

1. Go to **GitHub Repository** > **Settings** > **Secrets and variables** > **Actions** > **Variables** tab
2. Click **New repository variable** (or switch to the environment tab for per-environment)
3. Enter the **Name** and **Value**

### Environment-Specific Secrets

Some secrets differ between staging and production. These are set under:
**Settings** > **Environments** > select `staging` or `production` > **Environment secrets**

## Repository-Level Secrets

These are available to all workflows regardless of environment.

| Secret | Used By | How to Obtain | Rotation |
|--------|---------|---------------|----------|
| `GITHUB_TOKEN` | All workflows (auto-provided) | Automatically generated by GitHub Actions. No manual setup needed. | Automatic per-run |

> `GITHUB_TOKEN` is automatically available in every workflow run. It is used for GHCR login, creating releases, and posting PR comments. No manual configuration required.

## Staging Environment

Set these under: **Settings > Environments > `staging` > Environment secrets/variables**

### Secrets

| Secret | Used By | How to Obtain | Rotation |
|--------|---------|---------------|----------|
| `STAGING_SSH_KEY` | `deploy-staging.yml` | Generate an Ed25519 SSH keypair: `ssh-keygen -t ed25519 -f aion-staging -N ""`. Paste the **private key** content as the secret value. Add the public key to `~/.ssh/authorized_keys` on the staging VPS. | Rotate annually or when team members change |
| `STAGING_SSH_HOST` | `deploy-staging.yml` | The IP address or hostname of the staging VPS (e.g., `staging.aion.co` or `10.0.1.50`). | Update when VPS changes |
| `STAGING_SSH_USER` | `deploy-staging.yml` | The SSH username on the staging VPS (typically `root` or `deploy`). | Update when VPS changes |
| `STAGING_SUPABASE_URL` | `deploy-staging.yml` | Supabase Dashboard > Project Settings > API > **Project URL** for the staging project. Format: `https://xxxx.supabase.co` | Stable; changes only if Supabase project is recreated |
| `STAGING_SUPABASE_ANON_KEY` | `deploy-staging.yml` | Supabase Dashboard > Project Settings > API > **anon/public** key for the staging project. | Stable; regenerate in Supabase if compromised |

### Variables (non-secret)

| Variable | Used By | Value |
|----------|---------|-------|
| `STAGING_URL` | `deploy-staging.yml` (environment URL) | The public URL of the staging frontend (e.g., `https://staging.aion.co`) |
| `STAGING_API_URL` | `deploy-staging.yml` (smoke tests) | The public URL of the staging API (e.g., `https://staging.aion.co/api`) |
| `STAGING_GATEWAY_URL` | `deploy-staging.yml` (smoke tests) | The public URL of the staging gateway (e.g., `https://staging.aion.co/gw`) |

## Production Environment

Set these under: **Settings > Environments > `production` > Environment secrets/variables**

> **Important**: The production environment should have **Required reviewers** enabled (Settings > Environments > `production` > Deployment protection rules). Add at least one senior engineer as a required reviewer. Also set **Deployment branches** to "Selected branches and tags" and restrict to `v*` tags.

### Secrets

| Secret | Used By | How to Obtain | Rotation |
|--------|---------|---------------|----------|
| `PRODUCTION_SSH_KEY` | `deploy-production.yml` | Generate a **separate** Ed25519 keypair for production: `ssh-keygen -t ed25519 -f aion-production -N ""`. Add the public key to the production VPS `authorized_keys`. **Never reuse the staging key.** | Rotate annually or when team members change |
| `PRODUCTION_SSH_HOST` | `deploy-production.yml` | The IP address or hostname of the production VPS (e.g., `204.168.153.166` or `app.aionvisionhub.com`). | Update when VPS changes |
| `PRODUCTION_SSH_USER` | `deploy-production.yml` | The SSH username on the production VPS (typically `root` or `deploy`). | Update when VPS changes |
| `PRODUCTION_SUPABASE_URL` | `deploy-production.yml` | Supabase Dashboard > Project Settings > API > **Project URL** for the production project. Format: `https://xxxx.supabase.co` | Stable |
| `PRODUCTION_SUPABASE_ANON_KEY` | `deploy-production.yml` | Supabase Dashboard > Project Settings > API > **anon/public** key for the production project. | Stable; regenerate if compromised |

### Variables (non-secret)

| Variable | Used By | Value |
|----------|---------|-------|
| `PRODUCTION_URL` | `deploy-production.yml` (environment URL) | The public URL of the production frontend (e.g., `https://app.aionvisionhub.com`) |
| `PRODUCTION_API_URL` | `deploy-production.yml` (health checks) | The public URL of the production API (e.g., `https://app.aionvisionhub.com/api`) |
| `PRODUCTION_GATEWAY_URL` | `deploy-production.yml` (health checks) | The public URL of the production gateway (e.g., `https://app.aionvisionhub.com/gw`) |

## Secrets on the VPS (not GitHub)

These secrets live in the backend `.env` file on the VPS, **not** in GitHub. They are listed here for completeness as they are needed by the deployed application.

| Variable | Required | How to Obtain |
|----------|----------|---------------|
| `DATABASE_URL` | Yes | Supabase Dashboard > Project Settings > Database > **Connection string** (use the pooler connection on port 6543 for production). |
| `JWT_SECRET` | Yes | Must match the Supabase JWT secret. Found at: Supabase Dashboard > Project Settings > API > **JWT Secret**. Minimum 32 characters. |
| `CREDENTIAL_ENCRYPTION_KEY` | Production | Generate with: `openssl rand -hex 32`. Store securely; losing this key means encrypted device credentials cannot be decrypted. |
| `REDIS_URL` | Optional | If using Redis: `redis://localhost:6379` or the Redis provider's connection string. Falls back to in-memory cache if not set. |
| `CORS_ORIGINS` | Yes | Comma-separated list of allowed origins (e.g., `https://app.aionvisionhub.com,https://staging.aion.co`). |
| `OPENAI_API_KEY` | Optional | OpenAI Platform > API Keys > Create new secret key. Required for AI assistant features. |
| `ANTHROPIC_API_KEY` | Optional | Anthropic Console > API Keys. Required for Claude-based AI features. |
| `ELEVENLABS_API_KEY` | Optional | ElevenLabs Dashboard > Profile > API Keys. Required for voice intercom TTS. |
| `ELEVENLABS_DEFAULT_VOICE_ID` | Optional | ElevenLabs Dashboard > Voices > select voice > copy Voice ID. |
| `RESEND_API_KEY` | Optional | Resend Dashboard > API Keys. Use for email notifications (alternative to SMTP). |
| `SENDGRID_API_KEY` | Optional | SendGrid Dashboard > Settings > API Keys. Alternative email provider. |
| `SMTP_HOST` | Optional | Your SMTP server hostname (e.g., `smtp.gmail.com`). |
| `SMTP_PORT` | Optional | SMTP port (typically `587` for TLS, `465` for SSL). |
| `SMTP_USER` | Optional | SMTP authentication username. |
| `SMTP_PASS` | Optional | SMTP authentication password or app-specific password. |
| `EMAIL_FROM_ADDRESS` | Optional | Sender email address (e.g., `alerts@aionvisionhub.com`). |
| `WHATSAPP_PHONE_NUMBER_ID` | Optional | Meta Business Suite > WhatsApp > API Setup > Phone Number ID. |
| `WHATSAPP_ACCESS_TOKEN` | Optional | Meta Developers > App > WhatsApp > API Setup > generate System User token. |
| `WHATSAPP_BUSINESS_ACCOUNT_ID` | Optional | Meta Business Suite > Settings > WhatsApp Business Account ID. |
| `WHATSAPP_VERIFY_TOKEN` | Optional | A random string you choose when registering the webhook with Meta. Generate with: `openssl rand -hex 16`. |
| `WHATSAPP_APP_SECRET` | Optional | Meta Developers > App > Settings > Basic > App Secret. Used for webhook signature validation. |
| `EWELINK_APP_ID` | Optional | eWeLink Developer Portal > My Apps > App ID. Required for Sonoff/domotic control. |
| `EWELINK_APP_SECRET` | Optional | eWeLink Developer Portal > My Apps > App Secret. |
| `VAPID_PUBLIC_KEY` | Optional | Generate VAPID keypair: `npx web-push generate-vapid-keys`. Required for browser push notifications. |
| `VAPID_PRIVATE_KEY` | Optional | Generated alongside the public key above. |

## Setup Checklist

Use this checklist when configuring a new repository or environment:

### GitHub Repository Setup

- [ ] Repository created and code pushed
- [ ] Branch protection enabled on `main` (require PR review, require CI to pass)

### GitHub Environments

- [ ] `staging` environment created
- [ ] `production` environment created
- [ ] Production environment has **required reviewers** configured
- [ ] Production environment has **deployment branch policy** set to tags only (`v*`)

### Staging Secrets

- [ ] `STAGING_SSH_KEY` — private key for staging VPS
- [ ] `STAGING_SSH_HOST` — staging VPS IP/hostname
- [ ] `STAGING_SSH_USER` — SSH user on staging VPS
- [ ] `STAGING_SUPABASE_URL` — staging Supabase project URL
- [ ] `STAGING_SUPABASE_ANON_KEY` — staging Supabase anon key

### Staging Variables

- [ ] `STAGING_URL` — staging frontend URL
- [ ] `STAGING_API_URL` — staging API URL
- [ ] `STAGING_GATEWAY_URL` — staging gateway URL

### Production Secrets

- [ ] `PRODUCTION_SSH_KEY` — private key for production VPS (different from staging)
- [ ] `PRODUCTION_SSH_HOST` — production VPS IP/hostname
- [ ] `PRODUCTION_SSH_USER` — SSH user on production VPS
- [ ] `PRODUCTION_SUPABASE_URL` — production Supabase project URL
- [ ] `PRODUCTION_SUPABASE_ANON_KEY` — production Supabase anon key

### Production Variables

- [ ] `PRODUCTION_URL` — production frontend URL
- [ ] `PRODUCTION_API_URL` — production API URL
- [ ] `PRODUCTION_GATEWAY_URL` — production gateway URL

### VPS Configuration

- [ ] `.env` file created with all required variables (see table above)
- [ ] `.env` file permissions set to `600` (`chmod 600 .env`)
- [ ] SSH public keys added to `authorized_keys` on both VPS hosts
- [ ] Docker logged in to GHCR (`docker login ghcr.io`)

## Rotation Schedule

| Secret Type | Recommended Rotation | Procedure |
|------------|---------------------|-----------|
| SSH keys | Annually | Generate new keypair, update VPS `authorized_keys`, update GitHub secret |
| Supabase anon key | Only if compromised | Regenerate in Supabase Dashboard, update GitHub secret and VPS `.env` |
| JWT secret | Only if compromised | Change in Supabase Dashboard and VPS `.env` simultaneously; all active sessions will be invalidated |
| WhatsApp token | Every 60 days (temporary) or never (permanent) | Generate new token in Meta Developer Portal, update VPS `.env` |
| AI provider keys | Quarterly | Rotate in provider dashboard, update VPS `.env` |
| CREDENTIAL_ENCRYPTION_KEY | Never rotate (data loss) | Changing this key makes all encrypted device credentials unreadable. Only change if you re-encrypt all credentials first. |
| VAPID keys | Never rotate (breaks push subscriptions) | Changing VAPID keys invalidates all existing push subscriptions. Users would need to re-subscribe. |
